As the name suggests, working with this application is very simple. The xml document describing the finite state automata is very simple as well.

Here's the simplest thing you can describe using fsm-xml (trademark pending).

<fsm>
    <state id="0" type="f" label="initial">  
        <transition under="a">0</transition>  
    </state>  
</fsm>

Let's go through it step-by-step.

  1. The <fsm> element is the root element of the fsm-xml, it declares that the contents of the xml are describing a Finite State Machine (or Automaton). This attribute is mandatory

  2. The <state> element describes a state. The first state found in the xml document is always the initial state! On a graph, it will display as a node with a non-unique label, but the back-end of the app will always work with the unique id attribute.

  3. State's possible type attributes are as follows
    • r : this type is regular. A word contained in the language described by the automaton can not end in this state, but can pass through it.
    • f : this type is the final state of the machine. All words contained in the language generated by the automaton must end in this state

  4. The transition element can be nested inside a state attribute. It describes the letter under which does the word pass and to which state the destination is the id of an already existing state (or itself). On a graph this is represented with an edge.
